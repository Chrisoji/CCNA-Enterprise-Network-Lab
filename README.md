# CCNA Enterprise Network Lab

Welcome to my **CCNA Enterprise Network Lab** repository.

In this project, I built a complete set of labs focused on developing real, hands-on networking skills. The goal wasn’t just to study for the CCNA, but to truly understand how enterprise networks are configured and operate in practice.

This project also serves as a personal challenge to master Cisco CLI configurations and network administration while sharing that knowledge with others.

---

## What You’ll Find Here

Inside this repository, you’ll find **9 fully standalone labs**, each focused on a core CCNA topic.

I designed it this way so you can jump into *any* lab at any time without following a strict order. Every lab has its own topology, configurations, and explanations, so you can focus on exactly what you want to practice and improve.

| Lab | Topic                                      | Key Focus Area                  |
|-----|--------------------------------------------|---------------------------------|
| 01  | Basic Initial Configuration                | Device hardening & management   |
| 02  | VLANs and Trunking                         | Network segmentation            |
| 03  | Inter-VLAN Routing                         | Routing fundamentals            |
| 04  | Port Security                              | Layer 2 security                |
| 05  | DHCP Configuration                         | IP address services             |
| 06  | EtherChannel (LACP)                        | Link aggregation                |
| 07  | ACLs and NAT                               | Security + address translation  |
| 08  | Static Routing + OSPFv2                    | Routing protocols               |
| 09  | HSRP                                       | First-hop redundancy            |

---

## How to Navigate This Repository

Think of this README as your starting point.

- **`Labs/`** → Contains all lab environments  
  Each lab includes:
  - A dedicated `README.md` with objectives and step-by-step explanations  
  - Configuration files for reference or verification  
  - An `images/` folder with topologies and validation outputs  

- **`Addressing-scheme.md`** → A complete breakdown of the IP addressing used across all labs  

You can open any lab and start working immediately.

---

## How I Recommend You Use This

If you’re here to actually learn (not just skim), here’s the best approach:

1. Pick a lab based on the topic you want to practice  
2. Read the lab README to understand the objective  
3. Build the topology yourself in Packet Tracer  
4. Attempt the configuration on your own first  
5. Use my configs to verify or troubleshoot  
6. Run verification commands to confirm everything works  

That’s where the real learning happens.

## My Approach
